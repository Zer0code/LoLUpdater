Basically in LoLUpdater there are 4 files for the game and two files for the pvp.net client 3 of those game files are in one toolkit called "Cg", this program just updates the files to the latest version, then the other file called tbb.dll is very old in the game, so I got the source code for free and compiled it (created .dll from source files) in 5 different versions, each version of the tbb file is created for specific processors, first is the normal default one, then there is the normal one with SSE support (an SMID instruction), then there is one for SSE2, AVX and AVX2, in the lolupdater program it checks for what SMID instructions are supported and installs the tbb version with the highest supported version, lolupdater uninstalls PMB (malware), and applies the "EnableParticleOptimization" config tweak to the game, this is a config option that tweaks the lol engine, then it installs "DefaultParticleMultiThreading" support if your computer has 2 or more cores (lolupdater checks for this), then for the 2 pvp net files they are Adobe AIR.dll and "NPSWF32.dll" also called the Adobe Flash library, this again is outdated in lol so lolupdater attempts to bring these two .dll files to the latest version. It has built in stuff that autoupdates these dlls when updates are available. it only updates them if you run lolupdater, there is no service that is on all the time CG is provided by Nvidia and is now deprecated (will never update again), and for some reason Riot uses 3.0 when the latest is 3.1 Tbb is provided by Intel, and "Adobe Air" and "NPSWF32" are both Adobe files

LoLUpdater requires the .NET Framework 4.0
Place the file in your LoL directory and run it
